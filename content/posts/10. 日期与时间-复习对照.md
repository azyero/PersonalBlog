## 3. Date和Calendar

这一部分覆盖：10.2. Date和Calendar。

它的核心是：计算机表示的时间是以整数表示的时间戳存储的，即Epoch Time，Java使用`long`型来表示以毫秒为单位的时间戳，通过System.currentTimeMillis()获取当前时间戳。；Java有两套日期和时间的API：

- 相关概念：`String`、`变量`、`字节码`、`方法`、`包`、`LocalDateTime`、`ZonedDateTime`、`JDK`
- 代码关键词：`System.out.println`、`int`、`long`

**重点理解**

- 计算机表示的时间是以整数表示的时间戳存储的，即Epoch Time，Java使用`long`型来表示以毫秒为单位的时间戳，通过System.currentTimeMillis()获取当前时间戳。
- Java有两套日期和时间的API：
- 旧的Date、Calendar和TimeZone；
- 新的LocalDateTime、ZonedDateTime、ZoneId等。
- 分别位于java.util和 java.time包中。

**复习抓手**

- 标准库API
- Date
- Calendar
- TimeZone
- 小结

## 4. LocalDateTime

这一部分覆盖：10.3. LocalDateTime。

它的核心是：Java 8引入了新的日期和时间API，它们是不变类，默认按ISO 8601标准格式化和解析；；使用LocalDateTime可以非常方便地对日期和时间进行加减，或者调整日期和时间，它总是返回新对象；

- 相关概念：`LocalDateTime`、`ZonedDateTime`、`Instant`、`String`、`方法`、`JDK`
- 代码关键词：`long`

**重点理解**

- Java 8引入了新的日期和时间API，它们是不变类，默认按ISO 8601标准格式化和解析；
- 使用LocalDateTime可以非常方便地对日期和时间进行加减，或者调整日期和时间，它总是返回新对象；
- 使用isBefore()和 isAfter()可以判断日期和时间的先后；
- 使用Duration和 Period可以表示两个日期和时间的“区间间隔”。

**复习抓手**

- LocalDateTime
- DateTimeFormatter
- Duration和Period
- 小结

## 8. 最佳实践

这一部分覆盖：10.7. 最佳实践。

它的核心是：处理日期和时间时，尽量使用新的java.time包；；在数据库中存储时间戳时，尽量使用`long`型时间戳，它具有省空间，效率高，不依赖数据库的优点。

- 相关概念：`方法`、`Instant`、`ZonedDateTime`、`String`、`继承`、`LocalDateTime`
- 代码关键词：`long`

**重点理解**

- 处理日期和时间时，尽量使用新的java.time包；
- 在数据库中存储时间戳时，尽量使用`long`型时间戳，它具有省空间，效率高，不依赖数据库的优点。

## 易错点清单

- 旧的 `Date` 和 `Calendar` API 可变且设计复杂，优先使用 `java.time`。
- `LocalDateTime` 不含时区，跨地区场景应使用 `ZonedDateTime` 或 `Instant`。
- 格式化和解析要明确 pattern、Locale 和 Zone。
- 机器间传递时间更适合使用 `Instant` 或标准时间戳。
